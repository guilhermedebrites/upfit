## 3.3. Restrições Arquiteturais

As restrições impostas ao projeto que afetam sua arquitetura são:

* O sistema deve adotar uma arquitetura baseada em microserviços.
* O frontend web deve ser desenvolvido utilizando React.js.
* A aplicação mobile deve ser desenvolvida utilizando Flutter.
* O backend da aplicação deve ser desenvolvido utilizando Java com o framework Spring Boot.
* A comunicação entre clientes e os serviços do backend deve ocorrer por meio de APIs REST.
* O gerenciamento e exposição das APIs deve ser realizado utilizando Amazon API Gateway.
* A infraestrutura da aplicação deve utilizar serviços da plataforma AWS.
* Os microserviços devem ser executados em instâncias Amazon EC2.
* O armazenamento de dados deve utilizar Amazon RDS com PostgreSQL.
* O armazenamento de arquivos e imagens deve utilizar Amazon S3.
* A comunicação assíncrona entre microserviços deve utilizar Amazon SNS e Amazon SQS.
* O monitoramento e registro de logs da aplicação devem utilizar Amazon CloudWatch.
* O frontend da aplicação deve ser hospedado utilizando AWS Amplify.
* A resolução de nomes de domínio da aplicação deve utilizar Amazon Route 53.


## 3.4. Mecanismos Arquiteturais

A arquitetura do sistema UpFit é baseada em uma abordagem de microserviços, com comunicação síncrona via API Gateway e comunicação assíncrona por meio de mensageria utilizando SNS e SQS. O sistema é hospedado na infraestrutura da AWS, utilizando serviços gerenciados para garantir escalabilidade, disponibilidade e facilidade de manutenção.

| **Análise** | **Design** | **Implementação** |
| --- | --- | --- |
| Persistência | Separação de banco por domínio (Database per Service) utilizando banco relacional | Amazon RDS (PostgreSQL) com múltiplos bancos lógicos (auth_db, workout_db, group_db, challenge_db, progression_db) |
| Front end | Aplicação web com renderização no cliente e integração com APIs REST | Next.js hospedado no AWS Amplify |
| Back end | Arquitetura de microserviços com APIs REST e separação por domínio | Microserviços desenvolvidos em Java utilizando Spring Boot, executando em instância Amazon EC2 e expostos via Amazon API Gateway |
| Integração | Arquitetura orientada a eventos (Event-Driven Architecture) com comunicação assíncrona entre serviços | Amazon SNS para publicação de eventos e Amazon SQS para filas de consumo por microserviços |
| Log do sistema | Centralização de logs e monitoramento de serviços | Amazon CloudWatch Logs |
| Teste de Software | Testes unitários e testes de integração automatizados | Frameworks de teste da linguagem utilizada no backend (ex.: JUnit para serviços Java) e testes de API |
| Deploy | Integração contínua e implantação automatizada em infraestrutura AWS | AWS Amplify para deploy do frontend e deploy manual/automatizado dos microserviços na instância EC2 |
